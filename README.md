# buylow

**한국 주식(KOSPI·KOSDAQ) 자동매매 전략을 코드 없이 구성하고 과거 데이터로 백테스트하는 개인용 툴킷.**

[QuantConnect LEAN](https://github.com/QuantConnect/Lean) 엔진 위에서 동작하며, 동일한 전략 정의가 백테스트와 (추후) 실거래에서 그대로 실행됩니다.

<sub>현재 문서는 한국어로 제공됩니다. English / 日本語 번역은 추후 추가 예정입니다.</sub>

</div>

---

> **상태: 개발 중.** 전략 작성 · 데이터 적재 · 백테스트 · 리스크 관리까지 동작합니다.
> 실거래(토스증권) 연동은 토스 API 공개를 대기 중이므로, 현재는 백테스트·분석 도구로 사용하십시오.

## 개요

- 내 PC에서만 동작하는 로컬 웹 대시보드를 제공합니다. 브라우저로 `http://127.0.0.1:8420`에 접속하며, 시세 데이터와 API 키는 모두 로컬에만 저장됩니다(외부로 전송되지 않음).
- 전략은 **지표(시그널) 조합 규칙 + 리스크 설정**으로 정의합니다. `(EMA AND MACD) OR RSI`와 같은 매수 규칙을 체크박스로 구성하고, 손절·익절 등 리스크를 함께 지정합니다.
- 7가지 시그널을 제공합니다: 추세(EMA·MACD·모멘텀), 과열(RSI·볼린저밴드), 가치(저PER·저PBR), 한국 시장 특화 수급 추종(외국인·기관 순매수).
- 한국시장 전 종목 데이터로 백테스트를 실행하고, 결과를 한국어 요약과 거래 내역(매매 시점·종목·사유)으로 확인합니다.
- 공매도 없이(롱온리) 손절·익절·트레일링으로 포지션을 관리합니다.

## 아키텍처

```
브라우저 대시보드 (localhost:8420)
        │
Python 오케스트레이터 (상시 실행, FastAPI)
        │  작업마다 LEAN 프로세스를 생성
        ▼
LEAN 엔진 (.NET) — 전략을 백테스트(과거 재생) / (추후) 라이브로 실행
```

전략을 한 번 작성하면 백테스트와 실거래에 동일하게 사용됩니다(LEAN 동형성). 한국 시장 정의·수수료는 Python 레이어로, 실거래(토스) 연동은 별도 플러그인으로 결합됩니다. 상세 설계는 [docs/ARCHITECTURE.md](./docs/ARCHITECTURE.md)를 참고하십시오.

## 기능

### 1. 시그널(알파)

각 시그널은 매 거래일 종목별로 매수 우호 / 매도 우호 / 중립을 판단합니다. 전략 설정 탭에서 파라미터를 조정합니다.

| 시그널 | 유형 | 설명 | 주요 파라미터 |
|---|---|---|---|
| EMA | 추세 | 단기 이동평균이 장기 이동평균 위면 매수 우호, 아래면 매도 우호 | 단기·장기 기간 |
| MACD | 추세·모멘텀 | MACD선이 시그널선 위면 매수 우호 | 단기·장기·시그널 |
| RSI | 과열·역추세 | 과매도면 매수 우호, 과매수면 매도 우호 | 기간·과매도·과매수 |
| 모멘텀 | 추세 | 최근 N일 수익률이 양수면 매수 우호 | 관측 기간 |
| 볼린저밴드 | 평균회귀·돌파 | 밴드 터치 시 평균회귀, 강하게 돌파하면 추세 추종으로 전환 | 기간·표준편차 배수·전환 임계(%) |
| 저평가(가치) | 펀더멘털 | 저PER·저PBR이면서 ROE(=PBR/PER)가 기준 이상이면 매수 우호(가치 함정 회피) | PER·PBR 상한·ROE·배당 하한 |
| 수급 추종 | 한국 시장 특화 | 외국인·기관·개인(선택) 최근 N일 누적 순매수가 양수면 매수 우호 | 누적 일수·투자자 선택 |

가치·수급 시그널은 펀더멘털·수급 데이터가 적재되어 있어야 동작합니다(아래 *데이터 관리* 참고).

### 2. 매수 규칙

지표를 불리언 규칙으로 조합합니다. 대시보드의 조건 그룹 빌더에서 체크박스로 구성합니다.

- 한 그룹 안의 조건은 모두 충족(AND), 그룹끼리는 하나만 충족(OR)하면 매수합니다.
- 예: `[EMA + MACD]` 그룹과 `[RSI]` 그룹 → (EMA 그리고 MACD) 또는 (RSI).
- 신호 보유 기간: 매수 신호가 마지막으로 발생한 날로부터 며칠 더 보유할지 지정합니다. 매도 신호가 나오면 즉시 청산하고, 신호가 사라지면 이 기간 후 청산합니다.

전략은 하나만 저장되며(단일 전략), 변경 후 상단 저장 버튼으로 보관합니다.

### 3. 리스크 관리

매수는 전략이, 매도는 신호 변화와 아래 리스크 두 축으로 결정됩니다. 항목을 비우면 미적용입니다.

- 종목 손절(%): 매수가 대비 N% 하락 시 매도.
- 종목 익절(%): 평가이익 N% 도달 시 매도.
- 트레일링 스탑(%): 매수 후 고점 대비 N% 하락 시 매도(고점 자동 추적으로 이익 보호).
- 롱온리(공매도 차단)와 동시 보유 종목 상한(자본 대비 종목이 과다하면 유동성 상위만 보유)이 기본 적용됩니다.

### 4. 백테스트

- 기간 선택(날짜 선택기 및 1주·1달·3달·6달·1년 전 빠른 버튼).
- 유니버스 구성: 종목명·코드 검색 추가, 인덱스 일괄추가(KOSPI200·KOSDAQ150), 전체 종목 대상.
- 초기자본은 1억원으로 고정됩니다.
- 백그라운드로 실행되며 진행률과 로그를 작업중 탭에서 확인합니다. 실행 이력은 보관됩니다(SQLite).
- 결과는 한국어 요약(총 수익률·최종 자산·최대 낙폭·승률·샤프 지수 등)과 거래 내역(날짜·종목명·매수/매도·수량·금액·트리거 사유)으로 제공됩니다.

### 5. 데이터 관리

- 데이터 최신화 한 번으로 한국시장 전 종목의 가격(OHLCV)·수급(투자자별 순매수)·펀더멘털(PER/PBR)을 적재합니다.
  - 데이터 종류별로 마지막 적재일 다음날부터 오늘까지 증분 적재하며, 비어 있는 종류는 최근 5년을 백필합니다.
  - 가격은 무인증으로 받으며, 수급·펀더멘털은 KRX 로그인이 필요합니다.
- 종목명 자동 매핑(코드↔이름), 적재 현황 검색(이름·코드), 종목 상세 조회(가격·수급 전체 기간과 날짜 필터)를 제공합니다.

### 대시보드 구성

전략 설정(기본 화면) · 백테스트 · 데이터 · 설정(API 키) · 작업중(백그라운드 진행).

## 설치

### 사전 준비물

| 도구 | 용도 |
|---|---|
| [.NET 10 SDK](https://dotnet.microsoft.com/download) | LEAN 엔진 실행 |
| [Python 3.11](https://www.python.org/) | LEAN 전략(pythonnet)은 3.11이 필요합니다 |
| [uv](https://github.com/astral-sh/uv) | 파이썬 환경·패키지 관리 |
| git | 코드 내려받기 |
| (선택) [KRX 계정](https://data.krx.co.kr) | 수급·펀더멘털 데이터용(무료 가입). 가격만 사용할 경우 불필요 |

### macOS

```bash
# 1) 준비물 설치
curl -fsSL https://dot.net/v1/dotnet-install.sh | bash -s -- --channel 10.0 --install-dir "$HOME/.dotnet"
export DOTNET_ROOT="$HOME/.dotnet" && export PATH="$HOME/.dotnet:$PATH"   # 영구 적용은 ~/.zshrc에 추가
brew install python@3.11 git
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2) 코드 내려받기 + 의존성 설치
git clone https://github.com/JeongSeongMok/buylow.git
cd buylow
uv venv .venv && uv pip install --python .venv/bin/python -e ".[dev]"

# 3) 실행 후 브라우저에서 http://127.0.0.1:8420 접속
.venv/bin/python -m orchestrator.api
```

### Windows (PowerShell)

```powershell
# 1) 준비물 설치
winget install Microsoft.DotNet.SDK.10
winget install Python.Python.3.11
winget install Git.Git
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
# 설치 후 새 터미널을 열어 PATH를 반영합니다.

# 2) 코드 내려받기 + 의존성 설치
git clone https://github.com/JeongSeongMok/buylow.git

uv pip install --python .venv\Scripts\python.exe -e ".[dev]"

# 3) 실행 후 브라우저에서 http://127.0.0.1:8420 접속
.venv\Scripts\python -m orchestrator.api
```

### KRX 키 설정 (수급·펀더멘털 사용 시)

대시보드 설정 탭에서 KRX 아이디·비밀번호를 입력하거나, `config.example.yaml`을 `config.local.yaml`로 복사해 채웁니다. `config.local.yaml`은 저장소에 커밋되지 않습니다.

```yaml
# config.local.yaml
data_folder: ./data
dashboard_port: 8420
secrets:
  krx_id: ""     # https://data.krx.co.kr 무료 가입
  krx_pw: ""
```

## 사용 시작

브라우저로 `http://127.0.0.1:8420`에 접속한 뒤:

1. 데이터 탭에서 데이터 최신화를 실행합니다. 최초 1회는 필수이며(백테스트에 과거 데이터가 필요), 전 종목 적재이므로 작업중 탭에서 진행 상황을 확인합니다.
2. 전략 설정 탭에서 지표 값을 조정하고, 매수 규칙(조건 그룹)을 구성하고, 리스크를 설정한 뒤 저장합니다.
3. 백테스트 탭에서 기간과 종목을 선택하고 실행한 뒤, 결과와 거래 내역을 확인합니다.

## 파이프라인

```
데이터 최신화(가격·수급·펀더멘털) → ./data
        │
        ▼  (LEAN 5단계 · 매 거래일)
유니버스 → 시그널 판단(매수/매도) → 포트폴리오(롱온리·비중) → 리스크(손절·익절·트레일링) → 주문
        │
        ▼
백테스트(현재) = 과거 재생   /   라이브(예정) = 토스 실시간·주문
```

매수는 전략(시그널 규칙)이, 매도는 신호 변화와 리스크가 결정하는 두 축 구조입니다.

## 로드맵

**완료**
- LEAN 연동(NuGet + thin 런처, C#·Python e2e), KRX 시장 정의·수수료, 한국 시간대·무위험금리 보정
- 데이터 최신화 — 전 종목 가격·수급·펀더멘털, 데이터 종류별 증분, 종목명 매핑
- 규칙 엔진(조건 그룹 빌더) + 단일 전략 영속화 + 신호 보유 기간
- 시그널 7종 — EMA, MACD, RSI, 모멘텀, 볼린저밴드, 저평가(가치), 수급 추종
- 유니버스 — 종목 검색, 인덱스 일괄추가(KOSPI200/KOSDAQ150), 전체 종목
- 포트폴리오 — 롱온리(공매도 차단), 동시 보유 상한(유동성 기준)
- 리스크 관리(전역) — 종목별 손절·익절·트레일링
- 백테스트 — 백그라운드 실행·진행률·실행 이력, 한국어 결과 요약, 거래 내역(사유 포함)
- 로컬 대시보드(전략·백테스트·데이터·설정·작업), 설정 및 시크릿 관리

**예정**
- 토스증권 라이브(`TossBrokerage`·`TossDataQueueHandler`) — 토스 API 공개 대기
- 라이브 실행 엔진(주기 스케줄링) — 토스 의존
- 분봉 ETL(단타·변동성 돌파), 파라미터 최적화(스윕)
- OpenDART 재무, 뉴스·센티먼트, 유니버스 사전 필터, 커스텀 리스크
- 결과 차트, 알림, 다중 전략 저장, 배포 패키징, 라이선스 결정

## 면책 조항

이 소프트웨어는 교육 목적으로 제공됩니다. 자동 매매는 상당한 금융 리스크를 수반하며, 사용에 따른 책임은 전적으로 사용자에게 있습니다. 제작자는 어떠한 금전적 손실에도 책임지지 않습니다. 사용 시 증권사 API 약관 및 관련 법규를 반드시 준수하십시오. 또한 백테스트 결과는 과거 데이터 기반 추정이며 미래 수익을 보장하지 않습니다.
