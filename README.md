# buylow

**한국 주식(KOSPI·KOSDAQ) 자동매매 전략을, 코드 없이 대시보드에서 만들고 과거 데이터로 백테스트하는 개인용 툴킷.**

[QuantConnect LEAN](https://github.com/QuantConnect/Lean) 엔진 위에서 동작하며, **같은 전략이 백테스트와 (추후) 실거래에서 그대로** 실행됩니다.

<sub>English / 日本語 번역은 추후 추가 예정입니다.</sub>

</div>

---

> ⚠️ **상태: 개발 중.** 전략 작성 · 데이터 적재 · 백테스트 · 리스크 관리까지 **모두 동작**합니다.
> **실거래(토스증권)만 토스 API 오픈 대기 중**이라 아직 실매매용은 아닙니다. 백테스트 분석 도구로 사용하세요.

## 한눈에 보기

- 🖥️ **내 PC에서만 도는 로컬 웹 대시보드** — 브라우저로 `http://127.0.0.1:8420` 접속. 데이터·API 키는 전부 로컬에만 저장(서버로 안 나감).
- 🧩 **전략 = 지표(시그널) 조합 규칙 + 리스크** — 체크박스로 `(EMA 그리고 MACD) 또는 RSI` 같은 매수 규칙을 만들고, 손절·익절 등 리스크를 함께 설정.
- 📈 **7가지 시그널** — 추세(EMA·MACD·모멘텀), 과열(RSI·볼린저), 가치(저PBR), 한국 특화 **수급 추종**(외국인·기관 순매수).
- 🔁 **백테스트** — 한국시장 전 종목 데이터로 과거를 재생. 결과를 한국어 요약 + 거래 내역(언제·무엇을·왜 사고팔았는지)으로 확인.
- 🐂 **롱온리·종목별 리스크** — 공매도 없이, 손절/익절/트레일링으로 포지션 관리.

## 어떻게 동작하나 (구조)

```
브라우저 대시보드 (localhost:8420)
        │
Python 오케스트레이터 (상시 실행, FastAPI)
        │  작업마다 LEAN 프로세스를 띄움
        ▼
LEAN 엔진 (.NET) — 전략을 백테스트(과거 재생) / (추후) 라이브로 실행
```

전략을 한 번 작성하면 백테스트와 실거래에 **동일하게** 쓰입니다(LEAN 동형성). 한국 시장 정의·수수료는 Python으로, 실거래(토스)는 별도 플러그인으로 결합됩니다. 자세한 설계는 [docs/ARCHITECTURE.md](./docs/ARCHITECTURE.md).

## 기능 자세히

### 1. 시그널(알파) — 매매 신호의 부품

각 시그널은 매 거래일 종목마다 **매수 우호 / 매도 우호 / 중립**을 판단합니다. 전략 설정 탭에서 값(기간 등)을 조정합니다.

| 시그널 | 유형 | 설명 | 주요 파라미터 |
|---|---|---|---|
| **EMA** | 추세 | 단기 이동평균이 장기 위면 매수 우호, 아래면 매도 우호 | 단기·장기 기간 |
| **MACD** | 추세·모멘텀 | MACD선이 시그널선 위면 매수 우호 | 단기·장기·시그널 |
| **RSI** | 과열/역추세 | 과매도면 매수 우호, 과매수면 매도 우호 | 기간·과매도·과매수 |
| **모멘텀** | 추세 | 최근 N일 수익률이 +면 매수 우호 | 관측 기간 |
| **볼린저밴드** | 평균회귀+돌파 | 밴드 터치 시 평균회귀, 강하게 돌파하면 추세 추종으로 전환 | 기간·표준편차배수·전환임계% |
| **저평가(가치)** | 펀더멘털 | 저PER·저PBR + ROE(=PBR/PER) 기준 이상이면 매수 우호(가치 함정 회피) | PER·PBR 상한·ROE·배당 하한 |
| **수급 추종** | 한국 특화 | 외국인·기관·개인(선택) 최근 N일 누적 순매수가 +면 매수 우호 | 누적 일수·투자자 선택 |

> 가치·수급 시그널은 펀더멘털·수급 데이터가 적재돼 있어야 동작합니다(아래 *데이터 관리* 참고).

### 2. 매수 규칙 (조건 그룹 빌더)

지표들을 **불리언 규칙**으로 조합합니다. 대시보드에선 체크박스로:

- **한 그룹 안의 조건 = 모두 충족(AND)**, **그룹끼리 = 하나만 충족(OR)**.
- 예: `[EMA + MACD]` 그룹과 `[RSI]` 그룹 → **(EMA 그리고 MACD) 또는 (RSI)**.
- **신호 보유 기간**: 매수 신호가 마지막으로 뜬 날로부터 며칠 더 보유할지. 매도 신호가 나오면 즉시 청산, 신호가 사라지면 이 기간 후 청산.

전략은 **하나만** 저장되며(단일 전략), 변경 후 상단 **저장** 버튼으로 보관합니다.

### 3. 리스크 관리 (전역)

매수는 전략이, **매도는 신호 변화 + 아래 리스크** 두 축으로 이뤄집니다. 비우면 미적용.

- **종목 손절(%)** — 매수가 대비 N% 하락 시 매도.
- **종목 익절(%)** — 평가이익 N% 도달 시 매도.
- **트레일링 스탑(%)** — 매수 후 **고점 대비** N% 하락 시 매도(고점 자동 추적 → 이익 보호).
- **롱온리(공매도 차단)** + **동시 보유 종목 상한**(자본 대비 너무 많은 종목이면 유동성 상위만 보유) 기본 적용.

### 4. 백테스트 관리

- **기간** 선택(날짜 선택기 + 1주/1달/3달/6달/1년 전 빠른 버튼).
- **유니버스**: 종목명·코드로 검색해 추가 / **인덱스 일괄추가**(KOSPI200·KOSDAQ150) / **전체 종목 대상**.
- **초기자본 1억원 고정.**
- **백그라운드 실행** — 진행률(%)·로그를 *작업중* 탭에서 실시간 확인, 실행 이력 보관(SQLite).
- **결과**: 한국어 요약(총 수익률·최종 자산(억/만원)·최대 낙폭·승률·샤프 등) + **거래 내역**(날짜·종목명·매수/매도·수량·금액·트리거 사유).

### 5. 데이터 관리

- **데이터 최신화** 버튼 하나로 한국시장 **전 종목**의 가격(OHLCV)·수급(투자자별 순매수)·펀더멘털(PER/PBR)을 적재.
  - 종류별로 **마지막 적재일 다음날~오늘**만 증분(비어 있으면 최근 5년 백필).
  - 가격은 무인증, **수급·펀더멘털은 KRX 로그인 필요**.
- **종목명** 자동 매핑(코드↔이름), **적재 현황 검색**(이름·코드), **종목 상세**(가격/수급 전체 날짜 + 날짜 필터).

### 화면(탭) 구성

**전략 설정**(랜딩) · **백테스트** · **데이터** · **설정**(API 키) · **작업중**(백그라운드 진행).

## 설치 & 실행

### 사전 준비물 (공통)

| 도구 | 용도 |
|---|---|
| [.NET 10 SDK](https://dotnet.microsoft.com/download) | LEAN 엔진 실행 |
| [Python 3.11](https://www.python.org/) | LEAN 전략(pythonnet)은 3.11 필요 |
| [uv](https://github.com/astral-sh/uv) | 파이썬 환경·패키지 관리 |
| git | 코드 받기 |
| (선택) [KRX 계정](https://data.krx.co.kr) | **수급·펀더멘털** 데이터용(무료 가입). 가격(OHLCV)만 쓸 거면 불필요 |

### 🍎 macOS

```bash
# 1) 준비물 설치
curl -fsSL https://dot.net/v1/dotnet-install.sh | bash -s -- --channel 10.0 --install-dir "$HOME/.dotnet"
export DOTNET_ROOT="$HOME/.dotnet" && export PATH="$HOME/.dotnet:$PATH"   # (영구 적용은 ~/.zshrc에 추가)
brew install python@3.11 git
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2) 코드 받기 + 의존성 설치
git clone https://github.com/JeongSeongMok/buylow.git
cd buylow
uv venv .venv && uv pip install --python .venv/bin/python -e ".[dev]"

# 3) 실행 → 브라우저에서 http://127.0.0.1:8420
.venv/bin/python -m orchestrator.api
```

### 🪟 Windows (PowerShell)

```powershell
# 1) 준비물 설치 (winget)
winget install Microsoft.DotNet.SDK.10
winget install Python.Python.3.11
winget install Git.Git
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
# (설치 후 새 터미널을 열어 PATH 반영)

# 2) 코드 받기 + 의존성 설치

uv pip install --python .venv\Scripts\python.exe -e ".[dev]"

# 3) 실행 → 브라우저에서 http://127.0.0.1:8420
.venv\Scripts\python -m orchestrator.api
```

### KRX 키 설정 (수급·펀더멘털을 쓸 경우)

대시보드 **설정** 탭에서 KRX 아이디/비밀번호를 입력하거나, `config.example.yaml`을 `config.local.yaml`로 복사해 채웁니다. `config.local.yaml`은 git에 올라가지 않습니다.

```yaml
# config.local.yaml
data_folder: ./data
dashboard_port: 8420
secrets:
  krx_id: ""     # https://data.krx.co.kr 무료 가입
  krx_pw: ""
```

## 처음 사용하는 순서 (3단계)

브라우저로 `http://127.0.0.1:8420` 접속 후:

1. **데이터** 탭 → **데이터 최신화** 클릭. *(최초 1회 필수 — 백테스트엔 과거 데이터가 필요. 전 종목 적재라 시간이 걸리니 작업중 탭에서 진행 확인.)*
2. **전략 설정** 탭 → 지표 값 조정 → 매수 규칙(조건 그룹) 작성 → 손절·익절 등 리스크 설정 → **저장**.
3. **백테스트** 탭 → 기간·종목 선택 → **백테스트 실행** → 결과·거래 내역 확인.

## 파이프라인

```
데이터 최신화(가격·수급·펀더멘털) → ./data
        │
        ▼  (LEAN 5단계 · 매 거래일)
유니버스 → 시그널 판단(매수/매도) → 포트폴리오(롱온리·비중) → 리스크(손절·익절·트레일링) → 주문
        │
        ▼
백테스트(지금) = 과거 재생   /   라이브(예정) = 토스 실시간·주문
```

- **매수 = 전략(시그널 규칙)**, **매도 = 신호 변화 + 리스크** 두 축.
